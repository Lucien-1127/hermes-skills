# 每日 AI 接案快訊格式規範

> 來源：使用者提供 Prompt 規範
> 用於：`tw_freelance_crawler.py --briefing`

## 搜尋範圍（agent cron 用 web_search 補）

### 台灣平台
- 104 外包網、518 外包網、Tasker.tw
- ProjectMan、PRO360、Toby、SOHO 案件區
- Dcard 工作板、PTT Soho 板
- Facebook AI 接案/AI 工程師社團公開貼文

### 國際平台
- Upwork、Fiverr、Freelancer.com
- Toptal、Contra、We Work Remotely (AI/ML 分類)
- Indeed、LinkedIn Freelance (AI 標籤)

### 關鍵字組合
AI、LLM、GPT、Claude、prompt engineering、RAG、
自動化、AI 文案、AI 客服、AI 代理 (agent)、
n8n、Make、Zapier、Dify、Coze、
Stable Diffusion、Midjourney、AI 繪圖、
語音複製、AI 影片、AI 顧問、AI 培訓

## 蒐集任務

1. 當日新上架案件 Top 5（台灣優先，國際補充 1-2 筆）
2. 案件類型分布（文案/自動化/開發/顧問/培訓等占比）
3. 預算分布（低 <1 萬 / 中 1-5 萬 / 高 5-10 萬 / 企業級 >10 萬）
4. 今日熱門關鍵詞 Top 5（標筆數）
5. 新興需求識別（本週首次出現或顯著成長的領域）
6. 市場熱度比較（今日 vs 本週平均 vs 上週同日）
7. 重點推案 1 件 + 客觀分析（競爭度、預算合理性、技能匹配廣度）

## 硬規則（LLM 必須遵守）

1. 連結必須真實可點擊，查不到不要編造
2. 預算統一用 NT$；美元案件以當前匯率換算並標註（原 $X）
3. **禁用 Markdown 表格、程式碼區塊、深度縮排**（Telegram 顯示會亂）
4. emoji 限視覺區隔用途，每段 1-3 個
5. 繁體中文（台灣用語），技術術語保留英文（LLM、RAG、prompt 等）
6. 找不到的欄位寫「資料不足」，不編造
7. 投案分析用客觀第三人稱：
   - ✅ 「此案適合 X 類接案者」
   - ❌ 「建議你投」「符合你的技術棧」
8. 引用來源附在最後一段
